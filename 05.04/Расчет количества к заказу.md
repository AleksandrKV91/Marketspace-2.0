Заказ = 
  (Продажи_день × Коэф_сезонности × Дней_лог_плеча)   ← базовая потребность
  + Страховой запас (% от базы, например 15-20%)        ← буфер
  - Остатки_всего (FBO + FBS + МС + В пути)             ← уже есть
  - Ожидаемые поставки                                   ← если есть в пути
Где коэффициент сезонности = значение целевого месяца из wb_анализ ниш / среднее по всем месяцам.


Коэффициент сезонности — твой пример
На твоём примере: заказ в марте, поставка к июлю-августу, горизонт 60 дней продаж (июль + август).
Шаг 1. Нормализуем коэффициенты ниши — делим каждый на среднее по году:


Среднее = (12.3+12.7+11.2+9.8+8.9+9.1+13.2+14.7+11.8+6.3+4.7+5.3) / 12 = 9.97

Июль:   9.1  / 9.97 = 0.913
Август: 8.9  / 9.97 = 0.893
Март:   6.3  / 9.97 = 0.632  ← текущий месяц
Шаг 2. Корректируем наблюдаемые продажи на сезон текущего месяца:


Базовые продажи (нормализованные) = Продажи_31д / Коэф_марта
= Продажи_31д / 0.632
Это «очищенные» среднедневные продажи без влияния сезона.
Шаг 3. Считаем потребность на 60 дней (июль + август):


Потребность = Базовые_продажи × (Коэф_июль + Коэф_август) / 2 × 60 дней
Шаг 4. Вычитаем то что уже есть:


К заказу = Потребность - (FBO + FBS + МС_склад + В_пути)
Для товаров с нулевыми продажами сейчас (только 2-3 месяца в году):
Если продажи 31д = 0 но коэффициенты целевых месяцев > среднего — значит товар просто вне сезона сейчас. Тогда берём исторические продажи в пиковые месяцы из sheet1 (за прошлый год в те же месяцы) как базу.


1. БАЗОВЫЕ ПРОДАЖИ/ДЕНЬ
   = sheet1 продажи 31д / 31
   (все дни, включая OOS — показываем отдельно % дней OOS)

2. СКОРРЕКТИРОВАННЫЕ ПРОДАЖИ (очищенные от сезона)
   = Базовые / Коэф_текущего_месяца_ниши

3. ПОТРЕБНОСТЬ НА ГОРИЗОНТ ЗАКАЗА
   Горизонт = лог.плечо из зеленки (у кого нет — 30 дней по умолчанию)
   
   Берём месяцы которые покрывает горизонт заказа:
   - Если сейчас март, лог.плечо 60 дней → покрываем май-июнь
   - Потребность = Скорр_продажи × Среднее(Коэф_май, Коэф_июнь) × Горизонт_дней

4. СТРАХОВОЙ ЗАПАС
   = 15 дней (из СВОД col I «запас 15 дней»)
   Если товара нет в СВОД — 15 дней по умолчанию
   Страх_запас = Скорр_продажи × Коэф_целевого_месяца × 15

5. ИТОГОВЫЙ ЗАКАЗ
   = Потребность + Страх_запас - (FBO + FBS + МС_склад + В_пути)
   Минимум 0 (отрицательный заказ = не нужно заказывать)

6. ДЛЯ СЕЗОННЫХ С НУЛЁМ ПРОДАЖ СЕЙЧАС
   Если продажи 31д < 5 шт/день И коэф_целевых_месяцев > 1.3 × среднее →
   Берём продажи за тот же период прошлого года из sheet1 как базу

Уточнение по лог.плечу = время доставки
Это меняет всю формулу. Если лог.плечо = время доставки (например 100 дней для Китая), то нам нужно покрыть весь период от сейчас до прихода товара + ещё запас после прихода.
Правильная логика тогда:

Заказ делаем сегодня (март)
Товар придёт через лог.плечо = 100 дней → июль
Нам нужно покрыть:
  А) Продажи ДО прихода: 100 дней × текущий темп (март-июнь)
     — это покрывается ТЕКУЩИМИ остатками, не заказом
  Б) Продажи ПОСЛЕ прихода: на сколько дней делаем запас?

Уточнение по лог.плечу = время доставки
Это меняет всю формулу. Если лог.плечо = время доставки (например 100 дней для Китая), то нам нужно покрыть весь период от сейчас до прихода товара + ещё запас после прихода.
Правильная логика тогда:
Страховой запас (дни) = √(лог.плечо) × коэф_вариации_продаж
где коэф_вариации = стандартное отклонение продаж 31д / среднее продаж 31д

Смысл: чем нестабильнее продажи и чем длиннее доставка — тем больше буфер. Для стабильного товара (пеленки, коэф ≈ 0.1) и лог.плечо 10 дней → буфер ≈ 3 дня. Для нестабильного сезонного с лог.плечо 100 дней → буфер ≈ 30+ дней.


Финальная формула — фиксируем
ДАНО:
  Сегодня = март (месяц заказа)
  Лог.плечо = 100 дней (приход = июль)
  Горизонт = 60 дней (продажи после прихода = июль + август)

ШАГ 1. Продажи/день (из sheet1)
  базовые_31д = среднее продаж за последние 31 день

ШАГ 2. Очистка от текущего сезона
  коэф_текущий = коэф_ниши[март] / среднее_коэф_по_году
  база_норм = базовые_31д / коэф_текущий
  (если продажи 0 → используем продажи за тот же месяц год назад)

ШАГ 3. Потребность на горизонт 60 дней
  месяцы_горизонта = [июль, август]  (60 дней от даты прихода)
  коэф_целевой = среднее(коэф_ниши[июль], коэф_ниши[август]) / среднее_по_году
  потребность = база_норм × коэф_целевой × 60

ШАГ 4. Страховой запас
  σ = стд.откл. продаж 31д
  CV = σ / базовые_31д  (коэф. вариации)
  страховой_дни = √(лог.плечо) × CV
  страховой_шт = база_норм × коэф_целевой × страховой_дни

ШАГ 5. Что уже есть
  в_наличии = FBO_WB + FBS_пушкино + FBS_смоленск + МС_склад + в_пути + в_производстве

ШАГ 6. К заказу
  к_заказу = потребность + страховой_шт - в_наличии
  к_заказу = max(0, к_заказу)

Ключевые решения:
* Лог.плечо = время доставки → дата прихода = сегодня + лог.плечо
* Горизонт 60 или 90 дней — переключатель прямо в интерфейсе
* Продажи — три кнопки 7д / 14д / 31д, переключаются на лету
* OOS дни отображаются рядом с продажами
* При нулевых продажах — используются данные год назад
* В расширенной карточке видно каждый шаг расчёта
