Для реализации переключения тем в Next.js 16 с сохранением «стеклянного» эффекта из Behance, лучше всего использовать связку next-themes и переменные Tailwind. 

1. Установка и настройка провайдера 
Установите библиотеку: 

bash
npm install next-themes


Создайте файл components/ThemeProvider.tsx: 

tsx
"use client";
import { ThemeProvider as NextThemesProvider } from "next-themes";

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  return (
    <NextThemesProvider attribute="class" defaultTheme="dark" enableSystem>
      {children}
    </NextThemesProvider>
  );
}


Оберните children в app/layout.tsx в этот компонент. 

2. Обновление Tailwind Config 
Чтобы стили менялись автоматически, используйте префикс dark: для темных значений и стандартные для светлых:

theme: {
  extend: {
    colors: {
      background: 'var(--background)',
      surface: 'var(--surface)',
      border: 'var(--border)',

  }
}


В globals.css пропишите переменные:

  --surface: rgba(255, 255, 255, 0.03);
  --border: rgba(255, 255, 255, 0.08);

3. Рекомендации по стилю (Light Mode vs Dark Mode) 
Чтобы дизайн из Behance не потерял «премиальность» в светлой теме: 
1. Прозрачность (Opacity):
    * В Dark Mode мы используем bg-white/5 (белый на черном).
    * В Light Mode используйте bg-black/[0.02] (черный на белом) или чистый белый с сильной тенью. Эффект стекла в светлой теме лучше всего передается через backdrop-blur-md и shadow-lg.
2. Градиенты фона (Mesh Gradients):
    * В темной теме пятна должны быть яркими (primary/20).
    * В светлой теме сделайте их максимально пастельными (primary/5), иначе они будут выглядеть как «грязные пятна» на белом фоне.
3. Цвет текста:
    * Используйте text-slate-900 для заголовков и text-slate-500 для подписей в светлой теме. Чисто черный (#000) выглядит слишком агрессивно.
4. Графики (Recharts):
    * В светлой теме измените stopOpacity в градиенте под графиком с 0.3 на 0.1, чтобы он не перетягивал внимание. 

4. Кнопка переключения (Switcher) 
Разместите этот компонент в Sidebar или Header: 

tsx
"use client";
import { useTheme } from "next-themes";
import { Sun, Moon } from "lucide-react";

export function ThemeToggle() {
  const { theme, setTheme } = useTheme();

  return (
    <button
      onClick={() => setTheme(theme === "dark" ? "light" : "dark")}
      className="p-2 rounded-xl bg-surface border border-border hover:scale-110 transition-all"
    >
      <Sun className="h-5 w-5 rotate-0 scale-100 transition-all dark:-rotate-90 dark:scale-0" />
      <Moon className="absolute h-5 w-5 rotate-90 scale-0 transition-all dark:rotate-0 dark:scale-100" />
    </button>
  );
}

Чтобы эффект «стекла» из кейса на Behance не превратился в обычный серый блок при переключении на светлую тему, нужно инвертировать логику бликов и теней.
Вот адаптированный компонент GlassCard, который автоматически подстраивается под тему через Tailwind-префикс dark:.

1. Универсальный компонент GlassCard
Замените старый код компонента на этот. Основной секрет здесь в смене цвета границ и интенсивности размытия.

tsx
"use client";
import { motion } from "framer-motion";

interface GlassCardProps {
  children: React.ReactNode;
  className?: string;
  delay?: number;
}

export function GlassCard({ children, className = "", delay = 0 }: GlassCardProps) {
  return (
    <motion.div
      initial={{ opacity: 0, y: 20 }}
      animate={{ opacity: 1, y: 0 }}
      transition={{ duration: 0.5, delay }}
      whileHover={{ y: -4, transition: { duration: 0.2 } }}
      className={`
        relative overflow-hidden transition-all duration-300
        /* Светлая тема */
        bg-white/70 backdrop-blur-md border border-black/[0.05] shadow-[0_8px_30px_rgb(0,0,0,0.04)]
        /* Темная тема */
        dark:bg-white/[0.02] dark:backdrop-blur-xl dark:border-white/10 dark:shadow-glass
        rounded-2xl p-6 
        ${className}
      `}
    >
      {/* Специфический блик для Темной темы (скрыт в Светлой) */}
      <div className="absolute inset-0 bg-gradient-to-br from-white/[0.05] to-transparent pointer-events-none hidden dark:block" />
      
      {/* Едва заметный внутренний градиент для Светлой темы */}
      <div className="absolute inset-0 bg-gradient-to-b from-white/40 to-transparent pointer-events-none dark:hidden" />
      
      <div className="relative z-10">
        {children}
      </div>
    </motion.div>
  );
}



2. Стилизация элементов внутри карты
Чтобы контент выглядел органично в обеих темах, используйте следующие связки классов:
* Заголовки: text-slate-900 dark:text-white
* Вторичный текст: text-slate-500 dark:text-gray-400
* Иконки (фон): bg-primary/10 dark:bg-primary/20
* Разделители: border-black/[0.05] dark:border-white/10

4. Финальный совет по стилю «как в кейсе»
В светлой теме (Light Mode) для эффекта «Apple Style / Behance» критически важно качество теней.
* Вместо стандартного shadow-md используйте «слоистые» тени (мы добавили их в GlassCard выше через shadow-[0_8px_30px_rgb(0,0,0,0.04)]). Это дает эффект мягкого рассеянного света, а не грязного пятна под картой.
Теперь ваш дашборд будет выглядеть одинаково премиально и в 2 часа дня, и в 2 часа ночи.

2. Компонент формы (components/ui/GlassForm.tsx) 
Эта форма идеально впишется в ваш дизайн: у неё полупрозрачные инпуты, которые подсвечиваются при фокусе, и анимированные сообщения об ошибках.

"use client";
import { useForm } from "react-hook-form";
import { zodResolver } from "@hookform/resolvers/zod";
import * as z from "zod";
import { motion, AnimatePresence } from "framer-motion";
import { Loader2, Send } from "lucide-react";

// 1. Описываем схему валидации (Zod)
const transferSchema = z.object({
  email: z.string().email("Введите корректный email"),
  amount: z.coerce.number().min(10, "Минимальная сумма — $10").max(5000, "Лимит — $5000"),
  note: z.string().max(50, "Не более 50 символов").optional(),
});

type TransferForm = z.infer<typeof transferSchema>;

export function GlassForm() {
  const { register, handleSubmit, formState: { errors, isSubmitting } } = useForm<TransferForm>({
    resolver: zodResolver(transferSchema),
  });

  const onSubmit = async (data: TransferForm) => {
    // Имитация запроса к API
    await new Promise((resolve) => setTimeout(resolve, 2000));
    console.log("Данные отправлены:", data);
    alert("Перевод выполнен!");
  };

  return (
    <form onSubmit={handleSubmit(onSubmit)} className="space-y-5">
      {/* Поле Email */}
      <div className="space-y-2">
        <label className="text-xs font-medium text-slate-500 dark:text-gray-400 ml-1">Получатель</label>
        <input
          {...register("email")}
          placeholder="example@mail.com"
          className={`w-full bg-white/5 dark:bg-black/20 border ${errors.email ? 'border-red-500/50' : 'border-white/10'} 
            rounded-xl px-4 py-3 outline-none focus:border-primary/50 focus:ring-1 focus:ring-primary/20 transition-all text-sm`}
        />
        <ErrorMessage message={errors.email?.message} />
      </div>

      {/* Поле Сумма */}
      <div className="space-y-2">
        <label className="text-xs font-medium text-slate-500 dark:text-gray-400 ml-1">Сумма ($)</label>
        <input
          {...register("amount")}
          type="number"
          placeholder="0.00"
          className={`w-full bg-white/5 dark:bg-black/20 border ${errors.amount ? 'border-red-500/50' : 'border-white/10'} 
            rounded-xl px-4 py-3 outline-none focus:border-primary/50 transition-all text-sm`}
        />
        <ErrorMessage message={errors.amount?.message} />
      </div>

      {/* Кнопка отправки */}
      <button
        disabled={isSubmitting}
        className="w-full bg-primary hover:bg-primary/90 disabled:opacity-50 text-white font-medium py-3 rounded-xl 
          shadow-glow transition-all flex items-center justify-center gap-2 mt-4"
      >
        {isSubmitting ? <Loader2 className="animate-spin" size={18} /> : <><Send size={16} /> Отправить</>}
      </button>
    </form>
  );
}

// Вспомогательный компонент для анимации ошибки
function ErrorMessage({ message }: { message?: string }) {
  return (
    <AnimatePresence>
      {message && (
        <motion.p
          initial={{ opacity: 0, height: 0 }}
          animate={{ opacity: 1, height: 'auto' }}
          exit={{ opacity: 0, height: 0 }}
          className="text-[11px] text-red-400 ml-1"
        >
          {message}
        </motion.p>
      )}
    </AnimatePresence>
  );
}

2. Логика «Умного слияния» в Zustand (store/useDataStore.ts) 
Чтобы данные не удалялись, а дополнялись, нам нужен уникальный идентификатор (например, id или date). Zustand будет хранить текущую базу и объединять её с новыми строками из файла. 

tsx
import { create } from 'zustand';
import * as XLSX from 'xlsx';

interface DataRow {
  id: string | number;
  date: string;
  value: number;
  [key: string]: any;
}

interface DataState {
  rows: DataRow[];
  importExcel: (file: File) => Promise<void>;
}

export const useDataStore = create<DataState>((set, get) => ({
  rows: [], // Ваша база данных в браузере

  importExcel: async (file) => {
    const reader = new FileReader();
    
    reader.onload = (e) => {
      const data = e.target?.result;
      const workbook = XLSX.read(data, { type: 'binary' });
      const sheetName = workbook.SheetNames[0];
      const sheet = workbook.Sheets[sheetName];
      
      // Превращаем лист в массив объектов
      const newRows: DataRow[] = XLSX.utils.sheet_to_json(sheet);
      
      const currentRows = get().rows;

      // Умное слияние по ID: 
      // Если ID совпадает — обновляем, если нет — добавляем в конец
      const mergedRows = [...currentRows];
      
      newRows.forEach(newRow => {
        const index = mergedRows.findIndex(r => r.id === newRow.id);
        if (index > -1) {
          mergedRows[index] = { ...mergedRows[index], ...newRow }; // Обновляем
        } else {
          mergedRows.push(newRow); // Добавляем новое
        }
      });

      set({ rows: mergedRows });
    };
    
    reader.readAsBinaryString(file);
  },
}));

    XLSX.utils.book_append_sheet(workbook, worksheet, "UpdatedData");

    // Генерируем бинарный файл
    const excelBuffer = XLSX.write(workbook, { bookType: 'xlsx', type: 'array' });
    const finalData = new Blob([excelBuffer], { type: 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet' });
    
    // Сохраняем файл с текущей датой в названии
    saveAs(finalData, `Updated_Report_${new Date().toLocaleDateString()}.xlsx`);
  },
}));
Используйте код с осторожностью.

Итоговый стек технологий (Финал): 
Теперь ваш стек — это полноценная No-Code BI платформа: 
1. Frontend: Next.js 16 (Turbopack) + TypeScript.
2. Design System: Glassmorphism UI (Tailwind CSS + Framer Motion).
3. Data Engine: SheetJS (XLSX/XLSB) — чтение, парсинг и генерация файлов.
4. Database (Memory): Zustand — умное слияние (Merge) старых и новых данных по ключам.
5. Persistence: LocalStorage (через Zustand Persist) — данные не пропадают при закрытии браузера.
6. Visuals: Recharts — автоматическое обновление графиков при загрузке новых файлов.
