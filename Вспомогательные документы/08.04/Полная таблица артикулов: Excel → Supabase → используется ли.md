### Как названия артикулов ищутся в коде (поиск по заголовку)

Каждый парсер ищет колонку по тексту заголовка — не по номеру колонки. Это нужно знать, если заголовок вдруг изменится.

| Парсер | Что ищет (строки поиска) | Ищет для |
|---|---|---|
| `parseCatalog.ts` | `'артикул мс'`, `'артикул mc'`, `'артикул склада'` | → `sku_ms` |
| `parseCatalog.ts` | `'артикул wb'` | → `sku_wb` |
| `parseCatalog.ts` | `'артикул китай'` | → `sku_china` |
| `parseCatalog.ts` | `'номенклатура'`, `'название'` | → `name` |
| `parseSkuReport.ts` | `'sku'`, `'артикул мс'`, `'артикул склада'`, `'артикул'`, `'номенклатура'` | → `sku_ms` (col 0 fallback) |
| `parseSkuReport.ts` | *(WB артикул не ищется)* | — |
| `parseStock.ts` | `'sku'`, `'артикул wb'`, `'артикул вб'` | → `sku_wb` (col B fallback) |
| `parseABC.ts` | `'артикул склада'`, `'артикул мс'`, `'артикул mc'`, `'артикул'` | → `sku_ms` (col 0 fallback) |
| `parseChina.ts` | `'артикул склада'`, `'артикул'` | → `sku_ms` (col 0 fallback) |

**Вывод**: если в Excel файле заголовок колонки изменится с «Артикул МС» на что-то другое — парсер не найдёт ключевое поле и будет использовать fallback (col 0). Это может привести к тому, что данные запишутся под неправильным ключом.

### Поля dim_sku — что загружается и что используется

| Поле в Supabase | Excel название | Используется в дашборде | Где |
|---|---|---|---|
| `sku_ms` | Артикул МС / Артикул склада | ✅ | JOIN ключ для всех метрик |
| `sku_wb` | Артикул WB | ✅ | JOIN ключ для остатков; отображается как «SKU» в таблице |
| `sku_warehouse` | Артикул склада (если отдельно) | ❌ | Загружается, нигде не используется |
| `sku_china` | Артикул Китай | ❌ | Загружается, нигде не используется |
| `name` | Номенклатура / Название | ✅ | Отображается во всех таблицах |
| `brand` | Бренд | ✅ | Поиск в sku-table, отображение |
| `supplier` | Поставщик | ❌ | Загружается, не отображается |
| `country` | Страна | ❌ | Загружается, не отображается |
| `subject_wb` | Предмет WB | ✅ | Категория (fallback если нет category_wb) |
| `category_wb` | Категория WB | ✅ | Основная категория |
| `nds_pct` | НДС, % | ❌ | Загружается, не используется |
| `market_share` | Доля рынка | ❌ | Загружается, в NicheTab пока не подключено |
| `niche_appeal` | Привлекательность ниши | ❌ | Загружается, в NicheTab пока не подключено |
| `availability` | Доступность | ❌ | Загружается, не используется |
| `buyout_pct` | Процент выкупа | ❌ | Загружается, не используется |
| `avg_rating` | Средний рейтинг | ❌ | Загружается, не используется |
| `seasonality` | Сезонность (текст) | ❌ | Загружается, не используется |
| `season_start` | Старт сезона | ❌ | Загружается, не используется |
| `season_length` | Длина сезона | ❌ | Загружается, не используется |
| `top_month` | Топ месяц | ❌ | Загружается, не используется |
| `month_jan..month_dec` | Январь..Декабрь (коэффициенты) | ⚠️ | Загружается, используется в orders/route для сезонности (но расчёт не реализован) |
| `top_phrase` | Топ-фраза по объёму | ❌ | Загружается, не используется |

### Что отображается в дашборде как «SKU» / «Артикул»

В разных вкладках «артикул» показывается по-разному:

| Вкладка | Что показывается | Откуда |
|---|---|---|
| Аналитика по SKU (таблица) | `sku_wb` (число), если нет — `sku_ms` | `String(wb \|\| sku.sku_ms)` |
| Свод → ТОП-15 | `sku_wb` (число) | `dim_sku.sku_wb` |
| Логистика → таблица | `sku_wb` (число) | `dim_sku.sku_wb` |
| Цены → таблица | `sku_wb` (число) | `fact_price_changes.sku_wb` |
| SKU Modal | оба: `sku_wb` + `sku_ms` | `dim_sku` |
| Ниши | не показывается | — |
