# Спецификация вкладок

## 1. Обзор

### Фильтры (вверху)
- Категория (dropdown, мультивыбор)
- Статус: Все | Новинки | Не новинки
- Период: Calendar компонент

### KPI карточки (5 штук)
| Метрика | Формула | Дельта |
|---|---|---|
| Выручка | sum(revenue) за период | vs предыдущий такой же период |
| ЧМД | sum(chmd_5d × дни) | vs предыдущий |
| Расходы реклама | sum(ad_spend) | vs предыдущий |
| Средняя маржа | weighted avg(margin_rub/revenue) | vs предыдущий |
| ДРР | sum(ad_spend)/sum(revenue) | vs предыдущий |

### Алерты (кликабельные блоки)
Клик → переход на «Таблица SKU» с автофильтром

| Алерт | Условие | Severity |
|---|---|---|
| 🚨 STOP ADS | total_stock=0 AND ad_spend_5d>0 | critical |
| 🚨 OOS с потерями | total_stock=0 AND dpd_7>0 | critical |
| ⚠️ LOW STOCK | days_stock < log_pleche | warning |
| 💸 DRR > Маржа | drr_total > margin_pct | warning |
| 🚀 Потенциал роста | ctr > 3% AND cr_order < 1% | info |

### Графики
1. **Выручка + Расходы** — Area chart с двойной осью Y (левая = выручка, правая = расходы)
2. **Воронка по дням** — Line chart (CTR / CR корзины / CR заказа)

### Сравнение периодов
Таблица: текущий период vs предыдущий (аналогичная длительность)
Строки: Выручка | ЧМД | Расходы | ДРР | CTR | CR корзины | CR заказа

### По менеджерам
Таблица: Менеджер | Выручка | ЧМД | ДРР | SKU кол-во
Клик на строку → фильтрует Таблицу SKU по менеджеру

## 3. SKU Modal

Открывается из любой вкладки. Полноэкранный на мобиле, правая панель на десктопе.

### Заголовок
- Статусы: [OOS badge] [Маржа badge] [Score badge]
- Артикул МС + Название
- Категория · Предмет · Бренд · Поставщик · Дата появления

### Блок 1 — Метрики за период
| | Текущий | Предыдущий | Δ |
|---|---|---|---|
| Выручка | | | |
| ЧМД | | | |
| Расходы | | | |
| ДРР | | | |
| CTR | | | |
| CR корзины | | | |
| CR заказа | | | |

### Блок 2 — Воронка
CTR · CR корзины · CR заказа
CPC · CPM · CPO

### Блок 3 — Остатки
FBO WB · FBS Пушкино · FBS Смоленск · Комплекты · **Итого**
Дней до OOS (таблица) · Дней до OOS (расчётный)
Продажи ~шт/день

### Блок 4 — SKU Score
Число 0-100 + класс + расшифровка:
- Маржа: X/30
- ДРР: X/20
- Рост: X/15
- CR: X/15
- Остаток: X/20
Активные штрафы (если есть)

### Блок 5 — Рекомендации (авто)
- STOP ADS (если применимо)
- SCALE ADS +N% (если margin высокая и ДРР низкий)
- FIX CONTENT (если CTR высокий, CR низкий)
- ORDER STOCK (если days_stock < log_pleche)

### Блок 6 — Графики
- Выручка + Расходы по дням (двойная ось)
- Воронка по дням

### Блок 7 — Изменения цен
Таблица: Дата | Цена до | Цена после | Δ% | Δ CTR | Δ CR корзины | Δ CR заказа

### Заметка
Textarea + кнопка сохранить → `sku_notes`

## 7. Заказ товаров (OrderTab)

### KPI (верхняя строка)
- 🚨 Критично (остаток < 50% лог.плеча)
- ⚠️ Требует внимания (< лог.плеча)
- 📭 OOS с наличием спроса
- 📦 К заказу (расч.заказ > 0)
- 💰 Сумма заказа

### Фильтры
Поиск | Период продаж: 7/14/31 дней | Горизонт: 60/90 дней | Статус остатка | ABC класс
Переключатель: Показать только к заказу

### Таблица
SKU WB | Название | Статус | ABC | 31д продажи | OOS дней | Тренд | Наличие | Дней остатка | Лог.плечо | Расч.заказ | Заказ мен. | Δ | Маржа

Клик на строку → OrderModal

### Order Modal — 4 блока

**Блок 1 — Остатки**
FBO WB | FBS Пушкино | FBS Смоленск | Склад МС | В пути | В производстве | **ИТОГО** | Дата прихода | Итого в движении

**Блок 2 — Продажи**
7 дн | 14 дн | 31 дн | Тренд 14д | Год назад | CV вариация
Дней остатка сейчас
Недели (4 строки): Неделя N (дата-дата) | шт

**Блок 3 — Расчёт заказа**
- База: 31д продажи ÷ дней с остатком
- Коэффициент текущего месяца (из dim_sku.month_*)
- База ÷ коэф = очищенный спрос
- Логистическое плечо: X дней
- Дата прихода (ориентировочная)
- Горизонт по месяцам:
  | Месяц | Коэф | Потребность шт |
- Итого потребность
- Страховой запас
- Итого нужно
- Минус на руках (FBO+FBS+склад+в пути+в пр-ве)
- **Расчётный заказ** [формула]
- Заказ менеджера (редактируемое поле)
- Себестоимость × количество

**Блок 4 — ABC и финансы**
ABC (ЧМД/Выручка) | ABC (Выручка/Оборачиваемость)
Маржа% | Рентабельность | Оборачиваемость | Выручка (мес)
Цена WB | GMROI расч.
Сезонность ниши | Пик | Мини-график коэффициентов по месяцам

**Заметка** + **план продаж по месяцам** (Mar/Apr/May/Jun/Jul/Aug из Потребности Китай)

## 8. Обновление данных (UpdateTab)

Порядок загрузки (важен для корректного маппинга):
1. **Свод** — сначала, т.к. все остальные ссылаются на dim_sku
2. **АВС анализ**
3. **Потребность Китай**
4. **Таблица остатков**
5. **Отчёт по SKU**

Для каждого файла:
- Drag & Drop зона + кнопка "Выбрать файл"
- Прогресс-бар (animated)
- Статус последней загрузки: дата/время | кем | строк | статус (ok/error)
- Детали ошибки при hover/клике

**История загрузок** (из таблицы `uploads`):
Дата | Файл | Кем | Строк | Статус
