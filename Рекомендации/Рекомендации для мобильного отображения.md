Адаптация под телефон

Адаптация аналитического дашборда с 650 колонками под мобильные устройства — это вызов. Обычный скролл тут не поможет, нужна стратегия «умной трансформации» интерфейса.
В Next.js 16 и Tailwind мы реализуем это через скрытие второстепенных данных и перестроение графиков.
1. Трансформация Таблицы в «Карточки»
На мобильных устройствах (до 768px) мы полностью скрываем гигантскую таблицу и заменяем её на вертикальный список карточек с горизонтальной прокруткой только для дат.
tsx
// components/ui/MobileProductCard.tsx
export function MobileProductCard({ product }: { product: any }) {
  return (
    <GlassCard className="mb-4 p-4 lg:hidden">
      <div className="flex justify-between items-start mb-3">
        <div>
          <h4 className="font-bold text-white text-sm">{product.sku_sklad}</h4>
          <p className="text-xs text-gray-500 line-clamp-1">{product.metadata["Название"]}</p>
        </div>
        <div className="text-right">
          <span className="text-[10px] bg-primary/20 text-primary px-2 py-1 rounded-full uppercase">
            {product.metadata["Бренд"]}
          </span>
        </div>
      </div>
      
      {/* Мини-график последних 7 дней прямо в карточке */}
      <div className="h-16 w-full bg-white/5 rounded-lg mb-3">
         {/* Сюда вставляем упрощенный Sparkline график */}
      </div>

      <div className="flex gap-4 overflow-x-auto pb-2 custom-scrollbar">
        {Object.entries(product.sales).slice(-7).map(([date, val]) => (
          <div key={date} className="flex-shrink-0 text-center bg-white/5 p-2 rounded-lg min-w-[60px]">
            <p className="text-[10px] text-gray-500">{date}</p>
            <p className="text-xs font-bold text-white">{val} шт</p>
          </div>
        ))}
      </div>
    </GlassCard>
  );
}

3. Оптимизация Графиков (Recharts)
Главная проблема мобильных графиков — наложение подписей осей.
Решение: Скрываем подписи X-оси через одну или оставляем только первую и последнюю дату.
Интерактив: Используем allowDataOverflow={true}, чтобы пользователь мог "зумить" график пальцами.
tsx
<XAxis 
  dataKey="name" 
  hide={isMobile} // Скрываем ось на мобилках, оставляем только тултип
  minTickGap={30} // Увеличиваем расстояние между датами
/>


4. Мобильное Меню (Drawer)
Вместо бокового сайдбара используем «нижнюю шторку» или бургер-меню, которое выезжает через Framer Motion.

5. Специфические CSS-фишки для мобилок:
Touch-friendly: Все кнопки и инпуты должны иметь высоту минимум 44px (стандарт Apple).
No Hover: На мобильных нет наведения. Эффекты whileHover из Framer Motion нужно заменить на whileTap.
Safe Areas: Учет «челок» и нижних полосок на iPhone через padding-bottom: env(safe-area-inset-bottom).
Итог адаптации:
Десктоп: Мощная таблица с 650 колонками, детальные графики, боковой сайдбар.
Мобайл: Список "живых" карточек товаров, упрощенные сводки (Sparklines), нижняя навигация.


Для мобильной адаптации в стиле современных финтех-приложений (как в кейсах на Behance) мы внедрим Bottom Navigation (нижнюю панель). Это удобнее всего для управления большим пальцем.
Добавим также «умную» логику переключения между десктопным сайдбаром и мобильным меню.
1. Компонент мобильной навигации (components/ui/BottomNav.tsx)
Используем Framer Motion для анимации активного состояния и Lucide для иконок.
tsx
"use client";
import React from 'react';
import { motion } from 'framer-motion';
import { LayoutDashboard, BarChart2, FileUp, Settings } from 'lucide-react';
import { useStore } from '@/store/useStore'; // наше хранилище

const navItems = [
  { id: 'dashboard', icon: LayoutDashboard, label: 'Главная' },
  { id: 'analytics', icon: BarChart2, label: 'Аналитика' },
  { id: 'upload', icon: FileUp, label: 'Данные' },
  { id: 'settings', icon: Settings, label: 'Меню' },
];

export function BottomNav() {
  const [active, setActive] = React.useState('dashboard');

  return (
    <nav className="lg:hidden fixed bottom-0 left-0 right-0 z-50 px-6 pb-8 pt-4 bg-background/80 backdrop-blur-xl border-t border-white/10 shadow-[0_-10px_30px_rgba(0,0,0,0.3)]">
      <div className="flex justify-between items-center max-w-md mx-auto">
        {navItems.map((item) => {
          const isActive = active === item.id;
          return (
            <button
              key={item.id}
              onClick={() => setActive(item.id)}
              className="relative flex flex-col items-center gap-1 group"
            >
              {/* Анимированный фон активной кнопки */}
              {isActive && (
                <motion.div
                  layoutId="bottom-nav-active"
                  className="absolute -inset-2 bg-primary/10 rounded-2xl"
                  transition={{ type: 'spring', stiffness: 300, damping: 30 }}
                />
              )}
              
              <item.icon 
                size={22} 
                className={`relative z-10 transition-colors ${isActive ? 'text-primary' : 'text-gray-500'}`} 
              />
              <span className={`text-[10px] font-medium transition-colors ${isActive ? 'text-white' : 'text-gray-500'}`}>
                {item.label}
              </span>
            </button>
          );
        })}
      </div>
    </nav>
  );
}


2. Адаптивный Layout (app/layout.tsx)
Теперь объединим всё так, чтобы Sidebar был на компьютере, а BottomNav — на телефоне.
tsx
export default function DashboardLayout({ children }: { children: React.ReactNode }) {
  return (
    <div className="flex min-h-screen bg-background text-white">
      {/* Sidebar только для больших экранов */}
      <aside className="hidden lg:block sticky top-0 h-screen">
        <Sidebar />
      </aside>

      {/* Контент: на мобилках добавляем отступ снизу, чтобы BottomNav не перекрывал данные */}
      <main className="flex-1 p-4 md:p-8 pb-24 lg:pb-8">
        {children}
      </main>

      {/* Bottom Nav только для мобилок */}
      <BottomNav />
    </div>
  );
}


3. Мобильные оптимизации для графиков и таблиц
Чтобы данные читались, используйте эти правила в Tailwind:
Сжатие сетки: grid-cols-1 md:grid-cols-3 — превращает 3 колонки показателей в одну на телефоне.
Высота графиков: На мобильных ставьте h-[250px], на десктопе — h-[400px].
Табличный скролл: Добавьте overflow-x-auto к контейнерам таблиц. На iOS это обеспечит плавную инерционную прокрутку пальцем.
Рекомендации по стилю «Premium Mobile UX»:
Haptic Feedback: Если вы делаете приложение (PWA), добавьте легкую вибрацию при клике на кнопки навигации.
No Zoom: Чтобы на iPhone инпуты не «прыгали» при фокусе, убедитесь, что размер шрифта в них не меньше 16px (или используйте text-sm в Tailwind и добавьте метатег user-scalable=no).
Dark-only Glass: На мобильных экранах с OLED-матрицами (iPhone, Samsung) Glassmorphism выглядит потрясающе — черный цвет реально черный, а размытие создает глубокий объем.
